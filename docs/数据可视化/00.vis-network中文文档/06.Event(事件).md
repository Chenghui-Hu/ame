## 事件

这些事件由交互模块触发。它们与用户输入相关。

| 姓名                                                         | 属性                                  | 描述                                                         |
| ------------------------------------------------------------ | ------------------------------------- | ------------------------------------------------------------ |
| 由交互、选择、拖动等触发的事件                               |                                       |                                                              |
| click                                                        | **Object**                            | 当用户单击鼠标或触摸屏设备时激发。传递属性结构为的对象:{  nodes: [Array of selected nodeIds],  edges: [Array of selected edgeIds],  event: [Object] original click event,  pointer: {   DOM: {x:pointer_x, y:pointer_y},   canvas: {x:canvas_x, y:canvas_y}  } }这是所有事件通用的结构。特别是对于Click事件，将添加以下属性:{ ...  items: [Array of click items], }单击项的位置: {nodeId:NodeId}       // node with given id clicked on  {nodeId:NodeId labelId:0} // label of node with given id clicked on  {edgeId:EdgeId}       // edge with given id clicked on  {edge:EdgeId, labelId:0}  // label of edge with given id clicked onitems_array的顺序是z顺序递减。因此，要获取最顶部的项，请获取索引0处的值 |
| doubleClick                                                  | 与click一样                           | 。由于双击实际上是两次单击，因此将触发两次单击事件，然后是双击事件。如果不想在触发双击事件时使用单击事件，只需检查单击事件之间的时间间隔，然后再处理它们。. |
| oncontext                                                    | 与click一样                           | 当用户用鼠标右键单击画布时激发。默认情况下，鼠标右键不选择。如果需要，可以使用方法getNodeAt 选择节点 |
| hold                                                         | 与click一样                           | 当用户单击并按住鼠标或在触摸屏设备上单击并按住时激发。在这种情况下，还会触发一个单击事件。 |
| release                                                      | 与click一样                           | 在画布上绘制完成后激发。可用于在网络顶部绘制。               |
| select                                                       | 与click一样                           | 当用户操作更改了选择时激发。这意味着节点或边缘已被选择、添加到选择或取消选择。所有选择事件仅在单击并保持时触发。. |
| selectNode                                                   | 与click一样                           | 在用户选择节点时激发。                                       |
| selectEdge                                                   | 与click一样                           | 在用户选择边缘时激发。                                       |
| deselectNode                                                 | **Object**                            | 在用户取消选择某个节点（或多个节点）时激发。上一个选择是在上一个用户事件之前选择的节点和边的列表。传递属性结构为的对象:{  nodes: [Array of selected nodeIds],  edges: [Array of selected edgeIds],  event: [Object] original click event,  pointer: {   DOM: {x:pointer_x, y:pointer_y},   canvas: {x:canvas_x, y:canvas_y}   }  },  previousSelection: {   nodes: [Array of previously selected nodeIds],   edges: [Array of previously selected edgeIds]  } } |
| deselectEdge                                                 | 与deselectNode                        | 当用户取消选择某个边缘（或多个边缘）时激发。上一个选择是在上一个用户事件之前选择的节点和边的列表。. |
| dragStart                                                    | 与click一样                           | 启动拖动时激发。                                             |
| dragging                                                     | 与click一样                           | 在拖动节点或视图时激发.                                      |
| dragEnd                                                      | 与click一样                           | 拖动完成时激发.                                              |
| hoverNode                                                    | **{node: nodeId}**                    | 如果启用了选项interaction:{hover:true}并且鼠标悬停在节点上，则激发。 |
| blurNode                                                     | **{node: nodeId}**                    | 如果启用了选项interaction:{hover:true}并且鼠标移离了它之前悬停的节点，则激发。 |
| hoverEdge                                                    | **{edge: edgeId}**                    | 如果启用了选项interaction:{hover:true} 并且鼠标悬停在边缘上，则激发 |
| blurEdge                                                     | **{edge: edgeId}**                    | 如果启用了选项 interaction:{hover:true}并且鼠标移离了它之前悬停的边缘，则激发 |
| zoom                                                         | **Object**                            | 在用户放大或缩小时激发。属性告诉您缩放的方向。比例是一个大于0的数字，与network.get scale（）得到的数字相同。单击“放大”或“缩小”导航按钮触发时，传递的对象的指针属性将为空。调整属性结构为的对象:{  direction: '+'/'-',  scale: Number,  pointer: {x:pointer_x, y:pointer_y} } |
| showPopup                                                    | **id of item corresponding to popup** | 在显示弹出窗口（工具提示）时激发。                           |
| hidePopup                                                    | **none**                              | 在隐藏弹出窗口（工具提示）时激发.                            |
| ***\**事件触发了物理模拟。可用于触发GUI更新.\**\***          |                                       |                                                              |
| startStabilizing                                             | **none**                              | 稳定开始时发射。当拖动一个节点并重新启动物理模拟以再次稳定时，也会出现这种情况。稳定不一定意味着“没有表现”。 |
| stabilizationProgress                                        | **Object**                            | 达到更新间隔迭代次数的倍数时激发。这只发生在“隐藏”稳定中。传递属性结构为的对象:{  iterations: Number // iterations so far,  total: Number    // total iterations in options } |
| stabilizationIterationsDone                                  | **none**                              | 当“隐藏”稳定结束时触发。这不一定意味着网络是稳定的；也可能意味着已经达到了选项中定义的迭代次数. |
| stabilized                                                   | **Object**                            | 在网络稳定或调用stopSimulation()时激发。它所花费的迭代次数可以用来调整稳定网络所需的最大迭代次数。传递属性结构为的对象:{  iterations: Number // iterations it took } |
| ***\**事件触发器.\**\***                                     |                                       |                                                              |
| resize                                                       | **Object**                            | 当画布大小已调整时激发，可以通过容器div大小更改时的重绘调用、使用新值的setSize（）调用或使用新的宽度和/或高度值的setOptions（）。传递属性结构为的对象:{  width: Number   // the new width  of the canvas  height: Number   // the new height of the canvas  oldWidth: Number // the old width  of the canvas  oldHeight: Number // the old height of the canvas } |
| ***\**由呈现模块触发的事件。可用于在画布上绘制自定义元素.\**\*** |                                       |                                                              |
| initRedraw                                                   | **none**                              | 在重新绘制开始之前激发。此时模拟步骤已完成。可用于在开始绘制新框架之前移动自定义元素. |
| beforeDrawing                                                | **canvas context**                    | 在画布被清除、缩放并转换到查看位置之后，但在绘制所有边和节点之前激发。可用于在网络后面绘制。 |
| afterDrawing                                                 | **canvas context**                    | 在画布上绘制完成后激发。可用于在网络顶部绘制.                |
| ***\**事件被视图模块触发。\**\***                            |                                       |                                                              |
| animationFinished                                            | **none**                              | 动画完成时激发.                                              |
| ***\**事件通过配置模块触发。\**\***                          |                                       |                                                              |
| configChange                                                 | **Object**                            | 当用户更改配置程序中的任何选项时激发。Options对象可以与setOptions方法一起使用，也可以使用json.stringify（）进行字符串化。您不必手动将选项放入网络：这是自动完成的。可以使用事件在数据库中存储用户选项。 |
